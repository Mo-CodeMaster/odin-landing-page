# Odin Landing Page

This project is a recreation of The Odin Project's Landing Page assignment. The objective was to build a complete webpage from a provided design mockup using only HTML and CSS. The project emphasizes semantic HTML, Flexbox layouts, spacing, typography, and visual hierarchy while reinforcing the fundamentals of front-end web development.

Although relatively small in scope, this project provided valuable practice in translating a static design into a structured, maintainable webpage.

## Project Description

The landing page consists of several common website sections, including a header, hero section, information cards, testimonial, call-to-action, and footer. Each section was built using semantic HTML and styled with CSS to closely match the provided design.

The primary goal was not only to recreate the layout but also to develop a better understanding of page structure, alignment, spacing, and consistent styling without relying on CSS frameworks or JavaScript.

## Getting Started

### Prerequisites

* A modern web browser
* A code editor such as Visual Studio Code

### Installation

```bash
git clone https://github.com/Mo-CodeMaster/odin-landing-page.git

cd odin-landing-page
```

### Running the Project

Open `index.html` directly in your browser, or start a local development server:

```bash
python3 -m http.server 8000
```

Then visit:

```text
http://localhost:8000
```

## What I Learned

This project significantly improved my understanding of how HTML and CSS work together to create complete webpage layouts. I became more comfortable using Flexbox to align elements, organizing content into semantic sections, and applying consistent spacing throughout a page.

One of the biggest takeaways was realizing that good design often comes from small details such as balanced spacing, alignment, and typography rather than complex code. Rebuilding the project from scratch also reinforced my understanding of the concepts and helped improve my confidence in writing HTML and CSS without constantly referring to examples.

## Challenges Faced

Matching the provided layout required careful attention to spacing, sizing, and alignment. Achieving a balanced design using only HTML and CSS involved several rounds of refinement, particularly when positioning sections with Flexbox and maintaining consistent spacing across the page.

These challenges helped strengthen my debugging skills and encouraged me to write cleaner, more maintainable code.
